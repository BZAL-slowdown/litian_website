# AGENTS.md

## 项目目标

本仓库用于建设「力天空间设计」企业官网，要求包含：

- 前台企业网站：不少于 20 个简体中文内容页。
- 后台管理：无需改源码即可编辑前台内容。
- 业务功能：预约咨询、销售登记、员工资料、客户资料、员工登录、客户注册/登录。
- 部署目标域名：`litian.zbhaowen.top`。

## 技术栈

- 前端：React + Vite + React Router + CSS。
- 后端：Node.js + Express + SQLite。
- 登录鉴权：JWT。

## 开发规范

- 优先保持内容可配置化：前台文案、案例、公告、FAQ 等应来自后台 API 或种子数据。
- 不要把客户、员工、销售等隐私资料硬编码到前端。
- 后台接口新增或修改时，同步更新前端调用与 README。
- 所有新增页面应在 `server/src/content.js` 中有初始内容，且能通过后台编辑。
- 修改 UI 后请至少运行 `npm run build --workspace client`。
- 修改后端 API 后请运行 `npm run test --workspace server`。

## 固定协作流程

以后本项目每个任务默认采用“四个独立 Codex 线程协同 + 主窗口规划汇总”流程。

### 主窗口职责

- 只负责理解需求、提出关键问题、拆分任务、制定执行计划。
- 根据任务复杂度把任务发送给固定的四个独立线程，并等待代理完成。
- 汇总四个代理的结果，做最终取舍、排序和交付说明。
- 如需落地代码修改，主窗口负责合并、验证、提交与推送；不要跳过代理评审直接结束。

### 四个固定独立线程

1. 代码质量与代码修改代理
   - 线程标题：`力天代理｜代码质量与代码修改`
   - 线程 ID：`019e9b2e-fc66-7802-808e-dbb4ce1fa166`
   - 关注代码结构、实现质量、命名、重复逻辑、可读性、改动范围。
   - 输出应包含优先级、文件路径、行号、建议修改方式。

2. 错误与风险代理
   - 线程标题：`力天代理｜错误与风险`
   - 线程 ID：`019e9b2f-025c-7991-8913-0f7945e81aa7`
   - 关注 bug、运行失败、部署风险、安全风险、权限问题、数据异常。
   - 输出应包含复现方式、严重程度、影响范围、建议修复。

3. 页面审美与体验代理
   - 线程标题：`力天代理｜页面审美与体验`
   - 线程 ID：`019e9b2f-080b-7ec3-b0d6-7b08c26c9ce7`
   - 关注前台视觉、移动端、交互反馈、后台操作体验、文案可读性。
   - 输出应包含页面位置、体验问题、优先级、改进建议。

4. 可维护性代理
   - 线程标题：`力天代理｜可维护性`
   - 线程 ID：`019e9b2f-0d60-7993-b682-498fa27b2708`
   - 关注架构边界、模块拆分、配置、测试覆盖、长期扩展风险。
   - 输出应包含后续演进建议和优先级。

### 协同节奏

- 任务开始：主窗口先快速理解项目状态和用户目标，再把任务发送给四个固定线程。
- 代理执行：四个线程并行工作，但职责不要重叠。
- 结果汇总：主窗口等待全部代理完成后，按严重程度合并重复项。
- 落地修改：优先处理 P0/P1 问题，再处理影响上线或部署的问题。
- 验证交付：至少运行与改动相关的构建、测试或 lint，并在最终回复中说明结果。


## 默认后台账号

首次启动会读取 `.env`：

- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`

生产部署前必须修改默认密码和 `JWT_SECRET`。
